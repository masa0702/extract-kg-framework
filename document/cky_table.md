# CKY Table 生成と操作

このドキュメントでは、`src/cky_table.py` に実装された `CkyTable` クラスを中心に、CKY 表の生成方法や表示ユーティリティについて解説します。文節分割済みのデータから CKY 表を構築し、後続の依存解析やマッチング処理に利用する流れを理解することが目的です。

## 1. CKY 表の構造

CKY 表は 2 次元リストで表され、行・列ともに 1 始まりのインデックスを持ちます。対角線上には文節を表す辞書が配置され、それ以外のセルには依存解析の候補 `candidates` などが格納されます。

```text
[
  [0, 1, 2, 3, ...],      # ヘッダー行
  [1, {...}, 0, 0, ...],  # 文節1
  [2, 0, {...}, 0, ...],  # 文節2
  ...
]
```

文節辞書には以下のようなキーを持つデータが入ります。

- `id` : 文節番号
- `candidate` : 文節表層形
- `span` : 文字位置範囲 `(start, end)`
- `tokens` : 形態素列
- `upos`, `xpos` : 品詞タグ
- `token_span` : 各トークンのスパン

## 2. 表の生成

### `create_initializing_cky_table`

文節データのリストを受け取り、初期状態の CKY 表を作成します。対角線上に文節辞書を配置し、そのほかのセルは 0 で初期化されます。

```python
clauses = [
    ["太郎は", (1, 3), ["太郎", "は"], ["PROPN", "PART"], ["名詞", "助詞"], [(1,1),(2,2)]],
    ["学校へ", (4, 6), ["学校", "へ"], ["NOUN", "ADP"], ["名詞", "助詞"], [(4,4),(5,5)]],
]
mat = CkyTable.create_initializing_cky_table(clauses)
```

### `display_simple_cky_table`

CKY 表の内容を番号と文節表層形のみで簡潔に表示します。デバッグ用途に便利です。

### `display_multiline_cky_table`

辞書の内容を整形しつつ複数行に渡って表示します。横に長い CKY 表でも見やすく表示できます。

## 3. JSON 入出力

`process_json_to_cky_and_save` を使うと、依存解析結果を含む JSON から CKY 表を生成し、同じ JSON 構造に `dependency_table` として保存できます。これにより、前処理からマッチングまで一貫したデータ形式で扱うことができます。

## 4. 依存候補の格納

CKY 表の各セルには後続の解析で推定される依存候補 `candidates` を格納します。`candidates` は辞書のリストとして保存され、マッチング時に参照されます。

以上が `CkyTable` を用いた CKY 表生成と基本的な操作の概要です。

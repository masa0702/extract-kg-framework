# リファクタリング／改善タスクリスト

本ドキュメントは、現在の実装を保ったままメンテナンス性・再利用性を高めるためのリファクタリング候補を列挙します。実行仕様を変えない範囲での最適化・整理タスクをまとめています。

## 1. 設定・依存関係まわりの整理
- **ファイルパスとタイムアウトの設定外出し**: 入力CSVやAST pickle、出力ディレクトリ、GPU/CPUのスロット数・タイムアウト値が `src/main.py` 内でハードコードされています。CLI引数や設定ファイルに切り出し、環境ごとに上書きできる形にすることで再利用性と安全性を高められます【F:src/main.py†L54-L90】。
- **外部モデル依存の明示化**: BERT系モデルのロード有無を `try/except` で吸収しており、失敗理由が分かりづらい状態です。依存関係検査を明示し、モデル未配置時にはログで通知した上で選択的に無効化する初期化フローへ整理すると運用時のトラブルシュートが容易になります【F:src/bert_modules.py†L3-L28】。

## 2. 信頼性・例外処理の強化
- **CKY解析フェーズのエラーハンドリング分離**: `CKYAnalyzer.analyze_cky_table` 内で候補構築とモデル推論・ヒューリスティックが混在しているため、異常時に影響範囲を切り分けにくい構造です。セル初期化、依存推定、結果格納をステップ化し、例外時にセル状態を保持したままログできるようリファクタリングします【F:src/bert_modules.py†L30-L110】。
- **変数マッピング後処理の安全性**: 係り受け結果をフィルタする `clean_variable_mapping` は線形探索を重ねており、大規模入力で性能低下や意図しない部分一致が起きやすい構造です。文節表の索引化や厳密一致フラグを追加して、同等の動作をより安全に保つよう最適化します【F:src/main.py†L115-L139】。

## 3. コード品質・再利用性
- **重複ユーティリティの統合**: `MyUtility.most_common` が2か所に重複定義され、片方だけ戻り値の仕様が異なる状態です。共通の実装に統合し、戻り値契約を明文化したdocstringにそろえることで利用箇所の混乱を防げます【F:src/utils.py†L57-L85】【F:src/utils.py†L129-L153】。
- **ユーティリティの出力抑制**: `ensure_directory_exists` が存在確認時にも標準出力へメッセージを出しており、バッチ実行時にノイズとなります。ロガー経由でのデバッグ出力やサイレントモードの導入で既存の挙動を変えずに可観測性を向上できます【F:src/utils.py†L220-L225】。
- **定数・ユーティリティの分離**: `src/main.py` にパス生成・文字列操作などの共通ユーティリティが併存しているため、テストや再利用が困難です。関数をモジュール化して import する形に整理すると、パイプラインロジックとユーティリティの責務が分離されます【F:src/main.py†L54-L185】。

## 4. パフォーマンス・運用面
- **リテラル出現カウントの事前計算**: マッチング前のリテラル順序判定や頻度カウントが毎回 `find_all_occurrences` / `count_occurrences_in_span` で実行されています。文単位で前処理キャッシュを作り回すだけで動作を変えずにCPU負荷を低減できます【F:src/main.py†L148-L176】。
- **テーブル構築時の出力制御**: CKY表生成ユーティリティが標準出力に可視化結果を出す前提になっており、大量データ処理時にI/Oが支配的になる可能性があります。表示有無をフラグ化し、既定では無出力にすることで性能を保持できます【F:src/cky_table.py†L137-L199】。

## 5. テストと検証の拡充
- **モデル有無を跨いだ回帰テスト**: 既存テストはフィルタ周辺のスモールケースに限定されているため、BERTモデル有無・ヒューリスティックモード双方でのCKY候補生成を固定入力で回帰テスト化し、将来のリファクタリング時に挙動が変わらないことを保証します【F:src/bert_modules.py†L62-L109】【F:tests/test_literal_filter.py†L12-L38】。
- **設定値の境界テスト**: タイムアウト値や候補数上限などの設定が極端なデフォルトになっているため、最小値・最大値周辺の動作をユニットテストで確認し、想定外入力での動作変化を防ぎます【F:src/main.py†L82-L90】。
